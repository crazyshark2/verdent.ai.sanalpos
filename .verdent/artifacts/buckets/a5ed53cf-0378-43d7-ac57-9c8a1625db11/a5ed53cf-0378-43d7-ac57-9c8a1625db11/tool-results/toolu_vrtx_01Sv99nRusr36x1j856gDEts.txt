# Turkish Bank POS Gateway Architecture Analysis

Based on the page content, here's a comprehensive overview of the Turkish bank POS gateway system:

## **Architecture Overview**

The package provides a **unified interface class** (`PosInterface`) that enables the use of all Turkish bank virtual POS systems through a common API, eliminating the need to rewrite code when switching between different banks or payment models.

## **Gateway Classes**

Each bank is mapped to a gateway class in the configuration:

```php
'banks' => [
    'akbank' => [
        'name'  => 'AKBANK T.A.S.',
        'class' => Mews\Pos\Gateways\EstV3Pos::class,
        'gateway_endpoints' => [
            'payment_api'     => 'https://www.sanalakpos.com/fim/api',
            'gateway_3d'      => 'https://www.sanalakpos.com/fim/est3Dgate',
            'gateway_3d_host' => 'https://sanalpos.sanalakpos.com.tr/fim/est3Dgate',
        ],
    ],
]
```
